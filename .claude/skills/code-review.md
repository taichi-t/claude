レビュー対象: $ARGUMENTS

## 参照ドキュメント

`CLAUDE.md` のドキュメントセクションから以下を参照する。

- ディレクトリ構造・命名規則・型の書き方・テストの書き方・Lintルール
- レイヤー構成・モジュール間の依存方向・状態管理の責務分担・設計上の判断基準
- ブランチ戦略・PRの粒度・コミットメッセージ規約・レビュープロセス

以下の手順でコードレビューを行ってください。

## Step 1: コンテキストの収集

`/gather-context $ARGUMENTS` を実行してコンテキストを収集する。
何も渡されていない場合は、直近の git diff をレビュー対象とする。

## Step 2: レビュー

`CLAUDE.md` のドキュメントセクションに記載されたコーディング規約・コントリビューションガイド・アーキテクチャ方針のルールを基準としてレビューする。

問題点があれば具体的なファイル名・行番号とともに指摘し、修正案を提示する。
