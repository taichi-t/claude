$ARGUMENTS を解析し、コンテキストを収集する。
複数の入力が混在する場合はすべて処理する。

## 収集ルール

1. **URL が含まれる場合** → その URL に対応する MCP ツールを使って情報を取得する。利用可能な MCP ツールの中から最適なものを自動的に選択すること。
2. **ローカルファイルパスが含まれる場合** → Read ツールでファイルを読む。
3. **説明文・キーワードが含まれる場合** → そのまま要件・対象として扱う。
4. **何も渡されていない場合** → ユーザーに何を対象とするか質問する。

## 補足情報の収集

- `CLAUDE.md` のドキュメントセクションに記載されたコーディング規約・コントリビューションガイドを読み、ディレクトリ構造・依存関係・命名規則・ブランチ戦略を把握する
- `.claude/temp/` 配下に関連する仕様書・設計書がないか確認する
- 不明点があればユーザーに質問する
